## Overview

AutoDumper automatically finds all memory offsets and function addresses needed for game trainers/mods after each game update. Designed for UE4 games, currently configured for **Icarus**.

It combines 3 methods:

| Method | Source | What it finds |
|:-------|:-------|:-------------|
| **UE4SS Header Parser** | `CXXHeaderDump/*.hpp` | Class field offsets (Health, Stamina, etc.) |
| **PDB Symbol Loader** | `Game.pdb` via DbgHelp | Function addresses (SetHealth, ConsumeItem, etc.) |
| **AOB Scanner** | `Game.exe` binary | Byte pattern signatures (GWorld, code patches) |

## Output

Generates `offsets.json`:

```json
{
  "sdk": {
```

## Usage

```bash
# Run with defaults (auto-detects Icarus Steam path)
AutoDumper.exe

# Custom paths
AutoDumper.exe --game-path "C:\...\Win64" --dump-path "C:\...\CXXHeaderDump"
```

### Prerequisites

1. **UE4SS dump** must exist — inject UE4SS into the game and press `Ctrl+H` to generate `CXXHeaderDump/`
2. **PDB file** must be next to the game `.exe`

## How It Works

### 1. UE4SS Header Parser
Reads the C++ headers generated by [UE4SS](https://github.com/UE4SS-RE/RE-UE4SS) and extracts field offsets by pattern matching:
```cpp
// Finds lines like:
int32 Health;    // 0x1D8 (size: 0x4)
// Extracts: UActorState::Health = 0x1D8
```

### 2. PDB Symbol Loader
Uses Windows `DbgHelp` API (`SymEnumSymbols`) to enumerate all functions in the game's PDB and match them against a list of target functions.

### 3. AOB Scanner
Reads the `.text` section of the game binary and searches for known byte patterns to verify they still exist after updates.

## Adding New Games

Edit the `FIELD_QUERIES`, `FUNC_QUERIES`, and `AOB_QUERIES` arrays in `AutoDumper.cpp` to add offsets for other UE4 games.
